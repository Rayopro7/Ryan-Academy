Module 2: HTML Essentials

Welcome to Module 2! Now that we understand how the web works, let's dive into HTML (HyperText Markup Language). HTML is the standard language used to create and structure the content of web pages.

Lesson 2: Basic HTML Document Structure

Every HTML document follows a fundamental structure. Think of it like the skeleton of your webpage. Let's break down the essential parts you'll find in almost every HTML file.

1. The Document Type Declaration: ``

The very first line in any HTML document should be the DOCTYPE declaration.

<!DOCTYPE html>
  • Purpose: This tells the web browser which version of HTML the page is written in. For modern web development (HTML5), this specific declaration (``) is all you need.
  • Not an HTML Tag: It's an instruction for the browser about the document type.
  • Importance: Including this ensures the browser renders the page in "standards mode," following the latest web standards, which leads to more predictable and consistent results across different browsers. Without it, browsers might enter "quirks mode," trying to emulate older, buggier browser behavior, which you generally want to avoid.
  • Placement: It MUST be the very first thing in your HTML file, before the `` tag.

2. The Root Element: ``

Immediately after the DOCTYPE declaration comes the `` element. This element wraps all the content on the entire page.

<!DOCTYPE html>
<html>
  <!-- All other page content goes here -->
</html>
  • Role: It acts as the root container for the entire HTML document.
  • Attributes: Often includes the `lang` attribute to specify the primary language of the document's content, which helps search engines and assistive technologies (like screen readers). For English, you'd use: 
    <html lang="en">

3. The Head Element: ``

Inside the `` element, the first main section is the `` element. This section contains meta-information about the HTML document that is not directly displayed on the webpage itself (with a few exceptions like the ``).</p> <pre><code><!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>My Web Page Title</title> <!-- Other meta-info like links to CSS/JS files, descriptions, keywords go here --> </head> <!-- Body content comes next --> </html></code></pre> <ul> <li><strong>Content:</strong> Typically includes: <ul> <li><strong>`<meta>` tags:</strong> Provide metadata like character encoding (`charset="UTF-8"` is essential for displaying text correctly), viewport settings for responsive design, page descriptions, and keywords for SEO.</li> <li><strong>`<title>` tag:</strong> Defines the title that appears in the browser tab or window title bar, and is also used as the default name for bookmarks. The Body Element: `<body>`</h3> <p>Following the `<head>` section, still inside the `<html>` element, is the <strong>`<body>` element</strong>.
